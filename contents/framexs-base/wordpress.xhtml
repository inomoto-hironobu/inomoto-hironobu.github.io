Framexsはクライアントサイド駆動のフレームワークであるため、その動作は静的サイトか動的サイトかを問わない。

例えばWordPressでFramexsの機能を使えるテーマを作っている。

Framexs for WordPress
https://github.com/inomoto-hironobu/framexs4wordpress

Gitによってこれをインストールできます。

(wordpress)/wp-contents/theme/に移動して、下記のコマンドを実行する。

git clone https://github.com/inomoto-hironobu/framexs4wordpress.git

Framexsテーマを選ぶと管理画面でTheme Panelが選択できる。

このパネルではFramexsテーマのディレクトリを指定する。そうするとFramexsテーマが選べるようになる。

FramexsはWordPressのような動的サイトにどんな貢献があるだろうか？ Framexsを利用するためのWordPressテーマを用意しています。 テーマパネルで設定を行うことでFramexsテーマ機能を使うことができます。 例えばこのサイトで適用している八百幡テーマがWordPressでも使えるようになります。 FramexsテーマをWordPressテーマを使います。 このことによってHTML設計、CSS、JavaScriptのセットをWordPressの外に出すことができ、 例えば、スタティックサイトとWordPressサイトでUI、UXのひとまとめを共有することができます。 これはFramexsテーマの共有性を示すもので、サイト提供体制固有の記述に縛られることなく作製に専念できることを意味します。
